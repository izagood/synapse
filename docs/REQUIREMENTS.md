# Synapse 요구사항 정의서

> Markdown/HTML 기반, GitHub에 저장되는 Notion 스타일 노트 앱

## 1. 비전

- **편집은 Notion처럼, 저장은 Markdown으로.** 사용자는 블록 기반 WYSIWYG 에디터로 글을 쓰지만, 디스크에는 표준 `.md` 파일로 저장된다. 데이터는 언제나 사람이 읽을 수 있는 플레인 텍스트다.
- **AI 시대의 뷰어.** 요즘 AI가 정리해주는 결과물은 보기 좋은 HTML인 경우가 많다. Synapse는 `.html` 파일을 노트의 일급 시민으로 취급하여, 폴더 안의 HTML 문서를 바로 렌더링해서 볼 수 있다.
- **GitHub이 곧 저장소.** GitHub 로그인 한 번이면 노트 폴더가 GitHub 리포지토리와 동기화된다. 백업, 버전 관리, 기기 간 동기화를 모두 git으로 해결한다.
- **설정은 한 곳, 폴더는 자유롭게.** Obsidian은 vault(폴더)마다 설정이 따로 존재하지만, Synapse는 앱 전역 설정 하나만 둔다. 폴더는 VSCode처럼 "폴더 열기"로 아무 폴더나 즉시 열 수 있고, 폴더 안에 앱 설정 파일을 만들지 않는다.

## 2. 참조 제품과의 차별점

| 항목 | Obsidian | Notion | Synapse |
|---|---|---|---|
| 저장 형식 | `.md` (로컬) | 독점 DB (클라우드) | `.md` / `.html` (로컬 + GitHub) |
| 편집 경험 | 마크다운 소스 중심 | 블록 WYSIWYG | 블록 WYSIWYG (저장은 md) |
| 설정 위치 | vault마다 `.obsidian/` | 클라우드 계정 | 앱 전역 설정 1곳 |
| 폴더 열기 | vault 등록 필요 | 해당 없음 | VSCode식 즉시 열기 |
| 동기화 | 유료 Sync / 수동 git | 자체 클라우드 | GitHub 내장 동기화 |
| HTML 보기 | 미지원 | 미지원 | 내장 HTML 뷰어 |

## 3. 기능 요구사항

### FR-1. 폴더 / 파일 관리 (VSCode 모델)

- **FR-1.1** "폴더 열기"로 로컬의 임의 폴더를 워크스페이스로 연다. 사전 등록(vault 생성) 절차가 없다.
- **FR-1.2** 최근 연 폴더 목록을 제공하고, 시작 화면에서 바로 다시 열 수 있다.
- **FR-1.3** 파일 트리(사이드바)에서 파일/폴더 생성·이름 변경·이동·삭제를 지원한다.
- **FR-1.4** 빠른 열기(Quick Open, `Ctrl+P`)로 파일명 검색 후 즉시 이동한다.
- **FR-1.5** 전체 텍스트 검색(`Ctrl+Shift+F`)으로 폴더 내 노트 내용을 검색한다.
- **FR-1.6** 열었던 폴더에 Synapse가 설정 파일이나 숨김 폴더를 **생성하지 않는다** (git 동기화 메타데이터인 `.git/`은 예외).
- **FR-1.7** 탭으로 여러 문서를 동시에 열 수 있다.

### FR-2. 에디터 (Notion 스타일, Markdown 저장)

- **FR-2.1** 블록 기반 WYSIWYG 편집: 제목, 본문, 목록(순서/비순서/체크리스트), 인용, 코드 블록, 구분선, 표, 이미지.
- **FR-2.2** `/` 슬래시 커맨드로 블록 삽입 (Notion UX).
- **FR-2.3** 마크다운 단축 입력 지원: `# `, `- `, `> `, ``` 등 입력 시 즉시 해당 블록으로 변환.
- **FR-2.4** 저장 포맷은 CommonMark + GFM(표, 체크리스트, 취소선) `.md`. 라운드트립(열기→편집→저장) 시 원문 손실을 최소화한다.
- **FR-2.5** WYSIWYG ↔ 소스(raw markdown) 모드 전환 토글 제공.
- **FR-2.6** 자동 저장(디바운스) + 명시적 저장(`Ctrl+S`).
- **FR-2.7** 이미지 붙여넣기 시 워크스페이스 내 자산 폴더(설정 가능, 기본 `assets/`)에 저장하고 상대 경로로 링크한다.
- **FR-2.8** 내부 링크: `[[파일명]]` 위키링크와 표준 마크다운 상대 링크를 모두 인식하고 클릭 이동을 지원한다.
- **FR-2.9** YAML frontmatter는 파싱하여 보존하고, 속성 패널로 편집할 수 있다.

### FR-3. HTML 뷰어 (AI 산출물 열람)

- **FR-3.1** 파일 트리에서 `.html` 파일 클릭 시 렌더링된 결과를 탭에서 바로 보여준다.
- **FR-3.2** 렌더링은 샌드박스에서 수행한다: 외부 네트워크 요청 차단(옵션으로 허용), 스크립트 실행 기본 차단, sanitize 적용.
- **FR-3.3** 렌더 보기 ↔ 소스 보기 전환을 지원한다.
- **FR-3.4** HTML → Markdown 변환("md로 가져오기") 기능으로 AI가 만든 HTML을 편집 가능한 노트로 변환할 수 있다.
- **FR-3.5** 반대로 노트(.md)를 보기 좋은 HTML로 내보내기(export)할 수 있다.

### FR-4. GitHub 동기화

- **FR-4.1** GitHub OAuth(Device Flow) 로그인. 토큰은 OS 키체인에 저장한다.
- **FR-4.2** 새 워크스페이스를 GitHub 리포지토리(공개/비공개)로 게시하거나, 기존 리포지토리를 클론해서 열 수 있다.
- **FR-4.3** 자동 동기화: 변경 사항을 주기/이벤트 기반으로 commit하고 push/pull 한다 (커밋 메시지 자동 생성).
- **FR-4.4** 수동 동기화: 상태바의 동기화 버튼으로 즉시 push/pull.
- **FR-4.5** 충돌 처리: 충돌 발생 시 사용자에게 알리고, 양쪽 버전 비교 후 선택(또는 양쪽 보존)할 수 있는 UI를 제공한다.
- **FR-4.6** 오프라인 우선: 네트워크 없이도 모든 편집이 가능하고, 연결 복구 시 동기화한다.
- **FR-4.7** 파일 히스토리: git 로그 기반으로 특정 노트의 과거 버전을 보고 복원할 수 있다.
- **FR-4.8** git을 모르는 사용자도 쓸 수 있어야 한다 — 기본 UI에는 "동기화됨/동기화 필요/충돌"의 3가지 상태만 노출한다.

### FR-5. 설정 (전역 단일 설정)

- **FR-5.1** 모든 설정은 앱 전역 설정 한 곳(`~/.config/synapse/`)에 저장한다. 폴더별 설정 파일을 두지 않는다.
- **FR-5.2** 설정 UI는 하나의 설정 화면(검색 가능)으로 제공한다.
- **FR-5.3** 설정 항목(초기): 테마(라이트/다크/시스템), 에디터(폰트, 폰트 크기, 줄 간격, 자동 저장 간격), 자산 폴더 이름, 동기화(자동 동기화 on/off, 주기), HTML 뷰어 보안 수준, 언어.
- **FR-5.4** 설정 파일은 JSON으로 저장하여 사람이 직접 편집할 수도 있다.
- **FR-5.5** 워크스페이스별로 달라야 하는 최소한의 상태(연결된 리포지토리, 마지막 열림 탭 등)는 폴더가 아니라 **전역 설정 안의 워크스페이스 레지스트리**에 폴더 경로를 키로 저장한다.

### FR-6. 부가 기능 (post-MVP)

- **FR-6.1** 노트 간 백링크 패널.
- **FR-6.2** 그래프 뷰.
- **FR-6.3** AI 연동: 선택 영역 요약/정리, md→정돈된 HTML 변환 등 (사용자 API 키 기반).
- **FR-6.4** 모바일/웹 클라이언트 (GitHub을 백엔드로 공유).
- **FR-6.5** 플러그인 시스템.

## 4. 비기능 요구사항

- **NFR-1 (성능)** 10,000개 파일 워크스페이스에서 폴더 열기 3초 이내, 파일 열기 200ms 이내, 입력 지연 체감 없음.
- **NFR-2 (데이터 안전)** 앱이 크래시해도 작성 중 데이터가 유실되지 않는다. 파일 저장은 atomic write(임시 파일 후 rename)로 수행한다.
- **NFR-3 (호환성)** Synapse가 저장한 `.md`는 Obsidian, GitHub 웹 뷰어 등 표준 도구에서 그대로 읽혀야 한다.
- **NFR-4 (보안)** GitHub 토큰은 평문 저장 금지(OS 키체인). HTML 렌더링은 기본적으로 스크립트 차단 + sanitize.
- **NFR-5 (플랫폼)** Windows / macOS / Linux 데스크톱 지원.
- **NFR-6 (오프라인)** 모든 핵심 기능(편집, 검색, 열람)은 네트워크 없이 동작한다.
- **NFR-7 (용량)** 가벼운 설치 용량과 메모리 사용을 목표로 한다 (Tauri 선택 근거, ARCHITECTURE.md 참조).
