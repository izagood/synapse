# Synapse 요구사항 정의서

> Markdown/HTML 기반, GitHub에 저장되는 Notion 스타일 노트 앱 — AI 에이전트와 함께 쓰는 로컬 우선 워크벤치

이 문서는 살아있는 문서다. MVP(§5)는 완료되었고, 이후 기능은 이 문서에 계속
반영하며 지속적으로 개발한다. 코드 주석이 이 문서의 FR 번호(FR-1.3, FR-5.1 등)를
참조하므로 기존 번호는 의미를 바꾸지 않고 유지한다.

## 1. 비전

- **편집은 Notion처럼, 저장은 Markdown으로.** 사용자는 블록 기반 WYSIWYG 에디터로 글을 쓰지만, 디스크에는 표준 `.md` 파일로 저장된다. 데이터는 언제나 사람이 읽을 수 있는 플레인 텍스트다.
- **AI 시대의 뷰어이자 작업대.** AI가 정리해주는 결과물은 보기 좋은 HTML인 경우가 많다. Synapse는 `.html` 파일을 노트의 일급 시민으로 취급하여 폴더 안의 HTML 문서를 바로 렌더링해서 볼 수 있고, 나아가 내장 터미널과 MCP 브리지로 AI 코딩 에이전트가 노트 워크스페이스 안에서 직접 일하게 한다.
- **GitHub이 곧 저장소.** GitHub 로그인 한 번이면 노트 폴더가 GitHub 리포지토리와 동기화된다. 백업, 버전 관리, 기기 간 동기화를 모두 git으로 해결한다.
- **설정은 한 곳, 폴더는 자유롭게.** Obsidian은 vault(폴더)마다 설정이 따로 존재하지만, Synapse는 앱 전역 설정 하나만 둔다. 폴더는 VSCode처럼 "폴더 열기"로 아무 폴더나 즉시 열 수 있고, 앱 설정을 폴더 안에 두지 않는다(예외는 FR-1.6).
- **기능은 풍부하게, footprint는 가볍게.** "가벼운 앱"은 기능이 적다는 뜻이 아니다. 기능과 성능은 계속 늘리되 설치 용량·메모리·기동 시간을 가볍게 유지하는 것이 목표다(Tauri 선택 근거, NFR-7).

## 2. 참조 제품과의 차별점

| 항목 | Obsidian | Notion | Synapse |
|---|---|---|---|
| 저장 형식 | `.md` (로컬) | 독점 DB (클라우드) | `.md` / `.html` (로컬 + GitHub) |
| 편집 경험 | 마크다운 소스 중심 | 블록 WYSIWYG | 블록 WYSIWYG (저장은 md) |
| 설정 위치 | vault마다 `.obsidian/` | 클라우드 계정 | 앱 전역 설정 1곳 |
| 폴더 열기 | vault 등록 필요 | 해당 없음 | VSCode식 즉시 열기 |
| 동기화 | 유료 Sync / 수동 git | 자체 클라우드 | GitHub 내장 동기화 |
| HTML 보기 | 미지원 | 미지원 | 내장 HTML 뷰어 |
| AI 에이전트 연동 | 서드파티 플러그인 | 자체 클라우드 AI | 내장 터미널 + MCP 브리지 |
| 원격 폴더 | 미지원 | 해당 없음 | SSH/SFTP 원격 워크스페이스 |

## 3. 기능 요구사항

상태 표기가 없는 항목은 구현 완료된 현행 동작이다. 아직 구현되지 않은 항목은
**(후속)** 으로 표시한다.

### FR-1. 폴더 / 파일 관리 (VSCode 모델)

- **FR-1.1** "폴더 열기"로 로컬의 임의 폴더를 워크스페이스로 연다. 사전 등록(vault 생성) 절차가 없다.
- **FR-1.2** 최근 연 폴더 목록을 제공하고, 시작 화면에서 바로 다시 열 수 있다. 마지막 워크스페이스와 열려 있던 탭은 재시작 시 복원한다.
- **FR-1.3** 파일 트리(사이드바)에서 파일/폴더 생성·이름 변경·이동(드래그앤드롭)·복제·삭제를 지원한다. OS 파일 매니저로 드러내기, 외부 파일 드롭 가져오기, OS로 드래그 내보내기를 지원한다.
- **FR-1.4** 빠른 열기(Quick Open, `Ctrl+P`)로 파일명 검색 후 즉시 이동한다.
- **FR-1.5** 전체 텍스트 검색(`Ctrl+Shift+F`)으로 폴더 내 노트 내용을 검색한다.
- **FR-1.6** 워크스페이스 폴더에 앱 **설정**을 저장하지 않는다. 설정은 항상 전역(FR-5)에 있다. 단, 다음의 문서 데이터·연동 파일은 예외로 워크스페이스 안에 둘 수 있다:
  - `.git/` — git 동기화 메타데이터.
  - 노트가 참조하는 이미지 파일(FR-2.7).
  - `.synapse/draw/` — PDF 필기 사이드카(FR-10.2). 문서에 종속된 데이터이므로 문서와 함께 동기화되어야 한다.
  - `.mcp.json` — AI 에이전트용 MCP 브리지 연결 정보(FR-8.3). 생성 시 `.gitignore`에 자동 등록해 리포지토리를 오염시키지 않는다.
- **FR-1.7** 탭으로 여러 문서를 동시에 열 수 있다. 다른 탭 닫기/오른쪽 탭 닫기/모두 닫기를 지원한다. 다중 창을 지원한다.

### FR-2. 에디터 (Notion 스타일, Markdown 저장)

- **FR-2.1** 블록 기반 WYSIWYG 편집: 제목, 본문, 목록(순서/비순서/체크리스트), 인용, 코드 블록, 구분선, 표, 이미지.
- **FR-2.2** ~~`/` 슬래시 커맨드~~ — **채택하지 않음.** 블록 삽입은 마크다운 단축 입력(FR-2.3)으로 충분하다고 판단했다.
- **FR-2.3** 마크다운 단축 입력 지원: `# `, `- `, `> `, ``` 등 입력 시 즉시 해당 블록으로 변환.
- **FR-2.4** 저장 포맷은 CommonMark + GFM(표, 체크리스트, 취소선) `.md`. 라운드트립(열기→편집→저장) 시 원문 손실을 최소화하고, 손실 위험이 감지되면 저장을 차단한다.
- **FR-2.5** WYSIWYG ↔ 소스(raw markdown) 모드 전환 토글 제공.
- **FR-2.6** 자동 저장(디바운스, 간격 설정 가능) + 명시적 저장(`Ctrl+S`).
- **FR-2.7** 이미지 붙여넣기/드롭 시 **노트와 같은 폴더**에 저장하고 상대 경로로 링크한다. (별도 자산 폴더 설정은 두지 않는다 — 노트를 옮겨도 링크가 깨지지 않는 가장 단순한 규칙을 택했다.)
- **FR-2.8** 내부 링크: 표준 마크다운 상대 링크의 클릭 이동을 지원한다. `[[파일명]]` 위키링크(별칭 `[[파일명|표시명]]` 포함)는 백링크·그래프 뷰(FR-6)에서 인식한다. 에디터 안에서의 위키링크 렌더링·클릭 이동은 **(후속)**.
- **FR-2.9** YAML frontmatter는 파싱하여 보존한다. frontmatter 편집은 소스 모드에서 한다. (속성 패널 UI는 채택하지 않음.)
- **FR-2.10** 코드 블록 신택스 하이라이팅과 ```mermaid 다이어그램 렌더링을 지원한다.
- **FR-2.11** 문서 내 찾기(`Ctrl+F`)와 맨 URL 자동 링크화를 지원한다.
- **FR-2.12** 한글 등 CJK IME 조합 입력이 WebKit 웹뷰 안에서 안전하게 동작해야 한다(조합 중 트랜잭션 지연 처리).

### FR-3. HTML 뷰어 (AI 산출물 열람)

- **FR-3.1** 파일 트리에서 `.html` 파일 클릭 시 렌더링된 결과를 탭에서 바로 보여준다.
- **FR-3.2** 렌더링은 샌드박스에서 수행한다: iframe sandbox + DOMPurify sanitize, 외부 네트워크 요청 기본 차단(옵션으로 허용), 스크립트 실행 기본 차단(옵션으로 허용 — 이때 격리는 iframe sandbox가 담당).
- **FR-3.3** 렌더 보기 ↔ 소스 보기 전환을 지원한다.
- **FR-3.4** HTML → Markdown 변환(클립보드 HTML 가져오기 포함)으로 AI가 만든 HTML을 편집 가능한 노트로 변환할 수 있다.
- **FR-3.5** 노트(.md)를 단독 실행 가능한 HTML로 내보내기(export)할 수 있다.

### FR-4. GitHub 동기화

- **FR-4.1** GitHub OAuth(Device Flow) 로그인. 토큰은 전역 설정 디렉토리의 `secrets.json`(권한 0600)에 저장한다. macOS 키체인은 사용하지 않는다 — 셀프사인 빌드에서 키체인 ACL이 업데이트마다 무효화되는 문제를 피하기 위한 의도적 결정으로, `gh`/`aws` CLI와 같은 모델이다(구버전 키체인 항목은 1회 마이그레이션 후 정리). 시스템 git CLI를 동기화 엔진으로 사용한다.
- **FR-4.2** 새 워크스페이스를 GitHub 리포지토리(공개/비공개)로 게시하거나, 기존 리포지토리를 클론해서 열 수 있다.
- **FR-4.3** 자동 동기화: 설정한 주기마다 변경 사항을 commit하고 push/pull 한다(커밋 메시지 자동 생성, 실패 시 지수 백오프). 저장 이벤트 기반 즉시 동기화는 **(후속)**.
- **FR-4.4** 수동 동기화: 상태바의 동기화 인디케이터로 즉시 push/pull.
- **FR-4.5** 충돌 처리: 충돌 발생 시 사용자에게 알리고, 양쪽 버전 diff 비교 후 선택(내 것/원격/양쪽 보존)할 수 있는 UI를 제공한다. 자동 해소 가능한 충돌은 3-way 병합으로 자동 처리한다.
- **FR-4.6** 오프라인 우선: 네트워크 없이도 모든 편집이 가능하고, 연결 복구 시 동기화한다. git 미설치 상태도 우아하게 처리한다.
- **FR-4.7** 파일 히스토리: git 로그 기반으로 특정 노트의 과거 버전을 보고 복원할 수 있다.
- **FR-4.8** git을 모르는 사용자도 쓸 수 있어야 한다 — 기본 UI에는 "동기화됨/동기화 필요/충돌"의 3가지 상태만 노출한다.

### FR-5. 설정 (전역 단일 설정)

- **FR-5.1** 모든 설정은 앱 전역 설정 한 곳(`~/.config/synapse/`)에 저장한다. 폴더별 설정 파일을 두지 않는다.
- **FR-5.2** 설정 UI는 하나의 설정 화면으로 제공한다. 설정 화면 내 검색은 **(후속)**.
- **FR-5.3** 설정 항목: 테마(라이트/다크/시스템 + 커스텀 색), 언어(한국어/영어), 에디터(폰트, 폰트 크기, 자동 저장 간격, 줄 번호), 동기화(자동 on/off, 주기), HTML 뷰어 보안 수준(스크립트/네트워크), 삭제 확인, 터미널(외부 터미널 종류/커스텀 명령), 업데이트.
- **FR-5.4** 설정 파일은 JSON으로 저장하여 사람이 직접 편집할 수도 있다. 부분/손상 파일은 필드별 기본값으로 견고하게 로드한다.
- **FR-5.5** 워크스페이스별로 달라야 하는 최소한의 상태(연결된 리포지토리, 마지막 열림 탭 등)는 폴더가 아니라 **전역 설정 안의 워크스페이스 레지스트리**에 폴더 경로를 키로 저장한다.
- **FR-5.6** 설정 동기화: 전역 설정을 개인 git 리포지토리로 push/pull 하여 기기 간에 공유할 수 있다.

### FR-6. 링크 그래프 (백링크 · 그래프 뷰)

- **FR-6.1** 노트 간 백링크 패널: 현재 노트를 가리키는 위키링크/상대 링크를 수집해 보여준다.
- **FR-6.2** 그래프 뷰: 워크스페이스의 노트-링크 그래프를 시각화하고, 노드 클릭으로 이동한다.

### FR-7. 통합 터미널

- **FR-7.1** VS Code 스타일 내장 터미널 패널(`Ctrl+J`): PTY 기반, 워크스페이스 폴더에서 시작한다.
- **FR-7.2** WebKit 웹뷰 안에서 한글(CJK) IME 조합이 깨지지 않도록 조합 프리뷰 오버레이 등 안정화 레이어를 둔다.
- **FR-7.3** OS 네이티브 터미널(Terminal.app, iTerm2, Windows Terminal, 커스텀 명령)을 워크스페이스 폴더에서 바로 열 수 있다.

### FR-8. MCP 브리지 (AI 에이전트 연동)

- **FR-8.1** 실행 중인 Synapse가 로컬 TCP 브리지 서버를 열고, AI 코딩 에이전트가 MCP 서버(`synapse-mcp`)를 통해 접속해 현재 워크스페이스의 노트를 읽고 편집을 제안할 수 있다.
- **FR-8.2** 발견(discovery)은 `bridge.json` 파일로 자동화한다 — 수동 포트/토큰 설정이 필요 없다. 접속은 토큰으로 인증한다.
- **FR-8.3** 워크스페이스를 열 때 `.mcp.json`을 자동 프로비저닝해(번들 사이드카 존재 시) 에이전트가 즉시 연결할 수 있게 한다. `.gitignore`에 자동 등록한다(FR-1.6).
- **FR-8.4** 에이전트 편집과 사용자 편집이 충돌하면 병합 로직으로 안전하게 합친다.

### FR-9. 원격 워크스페이스 (SSH/SFTP)

- **FR-9.1** `ssh://` URI 또는 SSH config 호스트로 원격 서버의 폴더를 워크스페이스로 열 수 있다(SFTP 백엔드).
- **FR-9.2** 파일 트리·편집·저장이 로컬과 동일한 UX로 동작한다. 최근 폴더 목록과 세션 복원도 원격을 지원한다.
- **FR-9.3** 원격 워크스페이스의 git 동기화는 원격 호스트의 git을 SSH exec 채널로 구동한다.

### FR-10. 문서 뷰어 · 캔버스

- **FR-10.1** PDF 뷰어: `.pdf` 파일을 탭에서 열람(줌 지원).
- **FR-10.2** PDF 필기: PDF 위에 자유곡선 등으로 주석을 그리고, 원본을 건드리지 않는 사이드카(`.synapse/draw/….draw.json`)로 저장한다.
- **FR-10.3** draw.io 다이어그램(`.drawio`) 편집기 내장.
- **FR-10.4** Excalidraw 화이트보드(`.excalidraw`) 편집기 내장.
- **FR-10.5** 이미지 뷰어: 이미지 파일을 탭에서 열람(공용 줌 컨트롤).

### FR-11. 앱 업데이트

- **FR-11.1** 업데이트 확인과 원클릭 설치(tauri updater). 확인 주기는 설정에서 제어한다.

## 4. 비기능 요구사항

- **NFR-1 (성능)** 10,000개 파일 워크스페이스에서 폴더 열기 3초 이내, 파일 열기 200ms 이내, 입력 지연 체감 없음. 검색·Quick Open 결과에는 상한을 둔다. 대규모 트리 가상화 등 추가 장치는 실측 병목이 확인되는 지점부터 도입한다.
- **NFR-2 (데이터 안전)** 앱이 크래시해도 작성 중 데이터가 유실되지 않는다. 파일 저장은 atomic write(임시 파일 후 rename)로 수행한다.
- **NFR-3 (호환성)** Synapse가 저장한 `.md`는 Obsidian, GitHub 웹 뷰어 등 표준 도구에서 그대로 읽혀야 한다.
- **NFR-4 (보안)** GitHub 토큰은 권한 0600의 전용 파일에 저장하고(근거는 FR-4.1), 디스크 암호화 사용을 권장한다. HTML 렌더링은 기본적으로 스크립트 차단 + sanitize + 외부 네트워크 차단. MCP 브리지는 로컬 접속 + 토큰 인증.
- **NFR-5 (플랫폼)** 릴리스 바이너리는 macOS(.dmg, aarch64/x86_64)와 Windows(.msi)를 제공한다. Linux는 CI에서 빌드·테스트를 검증하며, 릴리스 패키징(AppImage/.deb)은 **(후속)**.
- **NFR-6 (오프라인)** 모든 핵심 기능(편집, 검색, 열람)은 네트워크 없이 동작한다.
- **NFR-7 (가벼움)** 기능이 많아져도 설치 용량·메모리 사용·기동 시간을 가볍게 유지한다. 기능 축소가 아니라 footprint 관리가 목표다: 시스템 웹뷰 재사용(Tauri), 무거운 임베드 에디터의 지연 로딩, 불필요한 상주 프로세스 금지.
- **NFR-8 (품질)** 기능에는 테스트가 따라온다(로직은 vitest/cargo test, jsdom으로 못 잡는 UI는 ladle/Playwright 시각 회귀). 모든 PR은 머지 시점에 릴리스 가능해야 한다(CI release-dry-run).
